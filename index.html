Introducing JSX

consider this variable declaration:

const element = <h1> Hello, world! </h1>;

This funny tag syntax is neither a string not HTML.

It is called JSX, and it is a syntax extension to JavaScript. We recommend using it with React to describe what the UI should look like. JSX may remind you of a template language, but it comes with the full power of JavaScript.

JSX produces React "elements". We will explores rendering them to the DOM in the next section. Below, you can find the basiscs of JSX necessary to get you started.

Why JSX?

React embraces the fact that rendering logic is inherently coupled with other UI logic: how events are handled, how the satate changes over time, and how data is prepared for display.

Instead of artificially separating technologies by putting markup and logic in separate files, React separate concerns with loosely coupled units called "components" that contain both. We will come back to components in a further section, but if you're not comfortable putting markup, this talk might convince you otherwise.

React doesn't require using JSX, buy most people find it helpful as a visual aid when working with UI inside JavaScript code. It also allows REact to show more usefur error and warnisng messages.

With that out of the way, let´s get started!

Rendering Elements

Elements are the smallest building blocks of React apps.

An element decribes what you want to see on the screen:

const element = <h1> Hello, world! </h1>;

Unlike browser DOM elements, React elements are plain objects, and are cheap to create. Reacta DOM takes care of updating the DOM to match the React elements.
